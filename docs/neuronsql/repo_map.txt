EXISTING APIs, CLIs, SERVERS
----------------------------
- HTTP: Gorilla Mux (no central router file; cmd/agent-server/main.go is missing).
- API routes (from handlers): /api/v1/agents, /api/v1/sessions, /api/v1/sessions/{id}/messages,
  /api/v1/llm/sql/generate, explain, optimize, debug, translate, models, /health, /metrics,
  /ws, workflows, memory, RAG, evaluations, tools, etc.
- CLI: neuronagent-cli (Cobra) in cli/main.go; bench and generate-key in cmd/.

EXISTING EMBEDDINGS OR RETRIEVAL CODE
--------------------------------------
- pkg/neurondb: embeddings, vector, RAG, hybrid search, reranking.
- internal/agent: knowledge_router.go (routes queries to knowledge sources), retrieval_learning.go,
  context.go (ContextLoader with retrieval).
- internal/tools: rag_tool.go, retrieval_tool.go, vector_tool.go, hybrid_search_tool.go.

EXISTING POSTGRES CONNECTION UTILITIES
--------------------------------------
- internal/db/connection.go: NewDB(connStr, poolConfig), retry with backoff, HealthCheck, GetPoolStats.
- internal/db/queries.go: CRUD for agents, sessions, messages, tools, memory, jobs, etc.
- Uses sqlx + lib/pq. Connection string from config (DatabaseConfig).

EXISTING CONFIG, LOGGING, AUTH PATTERNS
---------------------------------------
- Config: internal/config/config.go (Config struct with Server, Database, Auth, Logging, Workflow,
  Distributed, Cache, Multimodal). LoadConfig(path), LoadFromEnv, ValidateConfig.
- Logging: zerolog throughout.
- Auth: internal/api/middleware.go (AuthMiddleware with API key, principal, rate limiter).
  internal/auth: APIKeyManager, PrincipalManager, RBAC (RequireAnyRole, RoleAdmin, RoleUser).

WHAT WE WILL REUSE
-------------------
- internal/db: connection pooling, NewDB, HealthCheck (wrap with SafeConnection for NeuronSQL).
- internal/config: add NeuronSQLConfig; reuse LoadFromEnv pattern for neuronsql.*.
- internal/api: handler patterns (GetRequestID, GetAPIKeyFromContext, respondJSON, respondError,
  validation.ReadAndValidateBody), middleware stack.
- internal/tools: ToolHandler interface, Registry.RegisterHandler, ExecuteTool.
- pkg/llm_sql: types and validator patterns (we extend with pkg/neuronsql and internal/neuronsql).
- Gorilla Mux for route registration in cmd/agent-server/main.go.
- zerolog, OpenTelemetry (internal/observability), Prometheus (internal/metrics).

WHAT WE WILL ADD

- cmd/agent-server/main.go: HTTP server entry point, router, middleware, all handler registration.
- pkg/neuronsql: interfaces (LLMProvider, PolicyEngine, Retriever, Evaluator), models (NeuronSQLResponse).

- internal/neuronsql/tools: SafeConnection, schema_snapshot, validate_sql, explain_json, optimize,

- internal/neuronsql/eval: runner, metrics, suite, fixtures.

WHAT WE WILL NOT CHANGE

- internal/auth: API key, RBAC, audit (no changes).
- internal/db: existing queries and models (add only if needed for NeuronSQL).
